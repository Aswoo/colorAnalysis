# Color Analysis Implementation Guide (OpenCV + K-means)

이 문서는 `ColorAnalysisService`에서 구현된 색상 분석 로직을 단계별로 설명합니다.

---

## 1. 전제 조건 및 라이브러리 (JavaCV)

이미지 분석을 위해 **JavaCV (OpenCV Wrapper)** 라이브러리를 사용합니다. 
자바에서 C++로 작성된 OpenCV 기능을 편리하게 호출할 수 있게 해주는 도구입니다.

---

## 2. 코드 단계별 상세 설명

### **Step 1: 이미지 로드 (imdecode)**
*   사용자가 업로드한 `MultipartFile`은 바이트(byte) 형태입니다.
*   `opencv_imgcodecs.imdecode`를 사용하여 이 바이트 데이터를 OpenCV가 이해할 수 있는 **Mat(Matrix)** 객체로 변환합니다.

### **Step 2: 리사이징 (Resize)**
*   이미지의 픽셀이 너무 많으면 K-means 연산이 매우 느려집니다.
*   정확도에 큰 지장이 없는 수준인 **200x200** 크기로 줄여서 연산 속도를 획기적으로 높입니다.

### **Step 3: 데이터 구조 변경 (Reshape)**
*   이미지는 원래 (가로 x 세로 x 색상)의 3차원 구조입니다.
*   K-means 알고리즘은 픽셀들을 나열된 데이터로 처리해야 하므로, 이를 **1차원 배열 형태**로 길게 늘어뜨립니다.

### **Step 4: K-means 알고리즘 실행**
이 부분이 가장 핵심적인 **머신러닝(ML) 기반** 처리 단계입니다.
*   **K=5**: 이미지 내의 수많은 색상을 가장 대표적인 **5개**의 그룹으로 묶으라는 뜻입니다.
*   **centers**: 5개 그룹의 중심점, 즉 **대표 색상 5가지**의 RGB 값이 저장됩니다.

### **Step 5: 색상 판별 (Sentiment Match)**
*   추출된 5가지 대표 색상을 하나씩 검사합니다.
*   `isTargetColor` 메서드에서 사용자가 요청한 감성(GREEN, BLUE 등)과 일치하는 색이 하나라도 있는지 확인합니다.
*   **로직 예시 (Green)**: RGB 값 중 Green(G) 값이 Red(R)와 Blue(B)보다 월등히 높으면 초록색 느낌이 강하다고 판단합니다.

---

## 3. 주니어 개발자가 기억해야 할 점

1.  **Mat 객체는 행렬이다**: 이미지를 단순한 그림이 아니라 **숫자로 가득 찬 표(Matrix)**로 이해하는 것이 OpenCV의 시작입니다.
2.  **BGR 순서**: OpenCV는 표준 RGB 순서가 아니라 **BGR(Blue, Green, Red)** 순서로 색상을 저장합니다. (코드에서 `b`, `g`, `r` 순서로 값을 꺼내는 이유입니다.)
3.  **메모리 관리**: 대량의 이미지를 처리할 때 Mat 객체는 메모리를 많이 사용합니다. 실제 서비스 운영 시에는 사용이 끝난 Mat 객체를 해제해주는 것이 좋습니다.

---

## 4. 향후 고도화 계획 (HSV 도입)

현재 코드는 이해를 돕기 위해 단순한 RGB 비교 방식을 썼습니다. 
더 정확한 판별을 위해서는 `opencv_imgproc.cvtColor(image, hsvImage, COLOR_BGR2HSV)`를 사용하여 **HSV 색 공간**으로 변환한 뒤, 색상(Hue)의 각도 범위를 체크하는 방식으로 업그레이드할 수 있습니다.
